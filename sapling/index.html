Hush Sapling Upgrade

  • In repsonse to the Zcash Sprout inflation (CVE-2019-7167) bug, Hush will upgrade to Sapling. This will prevent attackers from potentially creating shielded funds, which they should not be able to. This is why it's called an "inflation" or "counterfeiting" bug, because both are happening. If an attacker can exploit the bug successfully they can fraudently create "new money" and also increase the total supply of the coin (inflation).
  • At this time, SEND ALL SHIELDED FUNDS to transparent addresses, to prepare for this upgrade. If all your funds are in an address starting with t1, no action is required.
  • If you have funds in a shielded (zaddr) which looks like zc..., YOU MUST TAKE ACTION.
  • Currently no cut-over time is set, but it will be decided soon. Shielded funds left in Sprout pool after a certain yet-undecided date WILL BE LOST. We must manage the security of the entire coin supply with the funds of each holder, so this time-frame has not yet been decided.
  • As of this writing Tue Feb 12 21:17:39 PST 2019, 1.2% of the current HUSH supply (71318.74047807 HUSH) is in INSECURE shielded sprout addresses.
